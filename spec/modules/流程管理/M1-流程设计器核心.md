# M1 流程设计器核心

## 模块概述

流程设计器是统一流程中台的建模入口，面向流程管理员提供可视化的拖拽式流程编排能力。模块支持 BPMN 2.0 标准，覆盖开始节点、审批节点、抄送节点、填写节点、条件/并行分支、子流程、结束节点等常见节点，并提供合法性校验、模板库、导入导出等辅助能力。

## 详细功能需求

### M1.1 可视化画布

**功能说明**
提供中心化的拖拽式画布，流程管理员可从左侧节点面板拖拽节点到画布，并通过连线建立节点关系。画布支持网格吸附、缩放（10% ~ 200%）、全屏编辑、自适应布局。

**业务规则**
- 画布最小尺寸 2000×2000 px，支持无限扩展；
- 节点拖拽时自动对齐网格（默认 10 px），可关闭吸附；
- 同一流程定义内节点 ID 全局唯一，连线 ID 全局唯一；
- 节点名称长度限制 2~50 个字符，不允许重名。

**前置条件**
用户已登录并具有「流程设计」权限；已选择流程分类并进入设计器。

**后置条件**
画布状态已保存到前端状态管理，可触发保存/发布动作。

**异常处理**
- 网络异常：保存失败时提示“保存失败，请检查网络后重试”，保留本地草稿；
- 非法拖拽：禁止将结束节点作为连线的起点；
- 节点重名：提示用户修改节点名称。

**界面原型描述**
顶部为工具栏（保存、撤销、重做、校验、发布、导入、导出、缩放比例、全屏），左侧为节点面板（分组：事件、任务、网关、子流程），中间为画布，右侧为属性面板。画布背景为点状网格，选中节点显示边框与拖拽手柄。

### M1.2 BPMN 2.0 标准支持

**功能说明**
流程定义以 BPMN 2.0 XML 作为底层存储格式，确保与 Camunda、Flowable、Activiti 等主流引擎的语义兼容。扩展属性通过 `flowable:` 命名空间或自定义 JSON 描述。

**业务规则**
- 必须支持 `bpmn:process`、`bpmn:startEvent`、`bpmn:userTask`、`bpmn:serviceTask`、`bpmn:exclusiveGateway`、`bpmn:parallelGateway`、`bpmn:subProcess`、`bpmn:endEvent`、`bpmn:sequenceFlow` 等元素；
- 自定义属性（如审批人、表单、超时配置）不得破坏 BPMN 2.0 标准语义；
- 导入外部 BPMN 文件时，无法识别的扩展属性放入扩展字段保留。

**前置条件**
流程定义需要持久化或导入导出。

**后置条件**
生成的 XML 可通过 BPMN 校验器验证基本语法。

**异常处理**
- 导入非法 XML：返回具体错误行号与原因；
- 缺少必要元素（如开始事件）：在校验阶段拦截。

**界面原型描述**
属性面板提供「BPMN XML」Tab，可预览/编辑原始 XML，编辑后需重新校验。

### M1.3 节点类型覆盖

**功能说明**
设计器提供以下节点：
- 开始节点：启动流程实例；
- 审批节点：需要指定审批人进行审批；
- 抄送节点：通知相关人员，不阻塞流程；
- 填写节点：指定人员补充表单数据；
- 条件分支节点：基于条件选择分支（排他网关）；
- 并行分支节点：多分支并行执行；
- 子流程节点：内嵌或调用外部流程；
- 结束节点：终止当前流程分支。

**业务规则**
- 一个流程定义必须且只能有一个开始节点；
- 结束节点可以有多个；
- 并行分支必须成对出现（分裂 + 汇聚）；
- 条件分支必须配置默认分支。

**前置条件**
画布已初始化。

**后置条件**
节点属性可配置，节点连线关系可用于合法性校验。

**异常处理**
- 删除唯一开始节点时提示“流程必须包含开始节点”；
- 并行分支未配对时校验失败。

**界面原型描述**
节点面板按图标+文字展示各节点类型，拖拽到画布后显示对应 BPMN 图标。

### M1.4 连接线与路由

**功能说明**
节点之间通过顺序流连接。条件流在排他/包容网关上使用，需配置条件表达式；默认流为兜底分支。连线支持直线、正交折线、贝塞尔曲线三种路由方式，并支持连线标签显示。

**业务规则**
- 顺序流起点只能是任务/网关/事件，终点不能是开始事件；
- 每个排他网关出度 ≥2，且必须有一条默认流；
- 条件流表达式基于流程变量，语法支持 SpEL/Ognl。

**前置条件**
画布上存在两个及以上节点。

**后置条件**
连线关系持久化到流程定义。

**异常处理**
- 连线重复：同一对节点之间允许存在多条条件不同的连线；
- 非法连接：开始事件不能作为终点；结束事件不能作为起点。

**界面原型描述**
选中连线后，属性面板显示「条件表达式」输入框和「默认分支」开关；连线上方可显示条件标签。

### M1.5 画布操作体验

**功能说明**
提供撤销/重做（操作栈深度 ≥ 50）、复制/粘贴（支持跨流程复制节点）、批量多选（框选 + Ctrl 点选）、快捷键绑定。

**业务规则**
- 快捷键：Ctrl+S 保存、Ctrl+Z 撤销、Ctrl+Y 重做、Ctrl+C/V 复制粘贴、Delete 删除；
- 粘贴时自动在鼠标位置生成副本，节点 ID 重新生成，名称加“-副本”。

**前置条件**
画布处于编辑状态。

**后置条件**
操作结果实时反映在画布与状态管理中。

**异常处理**
- 撤销栈为空时禁用撤销按钮；
- 批量复制包含非法节点时仅复制合法节点。

**界面原型描述**
工具栏提供撤销/重做按钮并显示可用状态；快捷键提示可在「帮助」菜单查看。

### M1.6 流程合法性校验

**功能说明**
保存/发布前自动执行校验，覆盖死循环、孤立节点、无出口分支、未设置审批人、节点名称缺失等。

**业务规则**
- 死循环：检测到节点通过连线可回到自身且无退出条件时报错；
- 孤立节点：没有入边也没有出边的节点（除开始/结束节点外）视为孤立；
- 未设置审批人：审批节点、填写节点必须配置办理人/候选人/候选组；
- 校验不通过时禁止发布。

**前置条件**
用户点击保存/发布。

**后置条件**
校验通过进入下一步；未通过展示错误列表并定位到画布对应节点。

**异常处理**
- 校验超时：异步校验，超时后提示“校验超时，请简化流程后重试”。

**界面原型描述**
右侧或底部弹出「校验结果」面板，列出错误项、警告项，点击后自动定位画布节点并高亮。

### M1.7 流程图运行时预览

**功能说明**
在流程实例详情页以只读方式展示流程图，已完成的节点使用绿色着色，当前处理节点高亮闪烁，待处理节点灰态，已完成路径使用箭头动画指引。

**业务规则**
- 颜色定义：已完成 #52c41a、进行中 #faad14 闪烁、待处理 #bfbfbf、异常 #f5222d；
- 支持显示每个节点的进入/离开时间、处理人、审批意见；
- 缩放与拖动与编辑画布一致。

**前置条件**
流程实例已创建且有历史任务数据。

**后置条件**
用户可直观查看流程运行轨迹。

**异常处理**
- 流程图与实例版本不一致时提示“当前流程定义已更新，图示为实例创建时的版本”。

**界面原型描述**
详情页上部分为表单摘要，下部分为流程图 Tab，图例说明节点状态颜色。

### M1.8 导入/导出能力

**功能说明**
支持将流程定义导出为 BPMN 2.0 XML 或 JSON 文件，也支持导入 BPMN XML 或历史 JSON 文件。

**业务规则**
- 导出 JSON 包含流程定义元数据、节点配置、表单配置、权限配置；
- 导入 BPMN XML 时，默认生成一个草稿版本；
- 导入后必须重新校验才能发布。

**前置条件**
用户具有导入/导出权限。

**后置条件**
导出文件下载到本地；导入后生成新的草稿流程定义。

**异常处理**
- 导入文件格式错误：提示“仅支持 .bpmn/.xml/.json 文件”；
- 导入版本不兼容：提示“文件版本 vX 与当前平台不兼容”。

**界面原型描述**
工具栏「导入」「导出」按钮，导出时弹窗选择格式。

### M1.9 流程模板库

**功能说明**
内置请假、报销、采购、入职、离职、借款、合同审批等通用模板，流程管理员可一键基于模板创建流程，也可将自定义流程保存为模板并共享。

**业务规则**
- 系统模板不可编辑，复制后生成可编辑副本；
- 自定义模板支持组织范围内共享或仅个人可见；
- 模板分类与流程分类体系一致。

**前置条件**
用户进入模板库页面。

**后置条件**
基于模板创建新的流程定义草稿。

**异常处理**
- 模板被删除：基于该模板创建的流程不受影响；
- 模板版本过旧：提示“模板基于旧版本规范，部分特性可能需要调整”。

**界面原型描述**
模板库以卡片形式展示模板缩略图、名称、分类、使用次数；支持搜索与分类筛选。

## 状态机/时序图描述

### 流程定义状态机

```text
[草稿] --发布--> [已发布] --停用--> [已停用]
   ↑                    |
   └------编辑生成新版本--┘
```

- 草稿：可编辑、可删除；
- 已发布：不可直接编辑，编辑后生成新版本草稿；
- 已停用：不可发起新实例，运行中实例继续执行。

### 保存/发布时序（文字描述）

1. 用户点击保存/发布；
2. 前端校验必填项；
3. 后端执行合法性校验；
4. 生成 BPMN XML 与扩展 JSON；
5. 保存流程定义版本；
6. 发布时同步写入 `bpm_definition_version` 并更新生效版本；
7. 返回成功/失败结果。
