# M4 流程控制与网关

## 模块概述

本模块负责流程的分支判断与路由，支持 BPMN 2.0 中的排他网关、并行网关、包容网关、事件网关，并提供条件表达式引擎、条件组合逻辑、条件优先级、默认分支、子流程、循环与回退能力。

## 功能架构图（文字描述）

```text
流程控制与网关
    │
    ├── 排他网关（XOR）
    ├── 并行网关（AND）
    ├── 包容网关（OR）
    ├── 事件网关
    ├── 条件表达式引擎
    ├── 子流程支持
    └── 循环与回退
```

## 详细功能需求

### M4.1 排他网关（XOR）

**功能说明**
排他网关在多条出流中选择第一条条件为 true 的分支执行，常用于金额区间、类型判断等场景。必须配置默认分支兜底。

**业务规则**
- 出度 ≥2；
- 条件按优先级顺序计算，命中后立即执行；
- 所有条件都不满足时走默认分支；
- 如果未配置默认分支且条件全部不满足，流程进入异常状态。

**前置条件**
网关上已配置至少两条出流及条件表达式。

**后置条件**
流程实例进入被选中的分支节点。

**异常处理**
- 多条条件同时为 true：按优先级顺序执行第一条；
- 无默认分支且条件全假：挂起并通知管理员。

**界面原型描述**
画布上以菱形「X」图标表示，属性面板列出出流条件与默认分支开关。

### M4.2 并行网关（AND）

**功能说明**
并行网关分裂为多条分支同时执行，所有分支完成后需经汇聚网关合并才能继续后续流转。

**业务规则**
- 并行分裂与汇聚必须成对出现；
- 所有进入汇聚网关的分支都完成后才继续；
- 并行分支内部可包含审批、填写等节点。

**前置条件**
流程中存在并行分裂网关与对应的汇聚网关。

**后置条件**
所有并行分支完成后激活汇聚网关后的节点。

**异常处理**
- 分支异常终止：整个流程实例异常挂起。

**界面原型描述**
菱形「+」图标，连线进入/离开网关。

### M4.3 包容网关（OR）

**功能说明**
包容网关根据条件选择所有满足条件的分支执行，无满足条件时走默认分支。

**业务规则**
- 可同时命中多条分支；
- 所有命中分支完成后经汇聚网关合并；
- 默认分支开关可配置。

**前置条件**
出度 ≥2 且已配置条件表达式。

**后置条件**
激活所有满足条件的分支任务。

**异常处理**
- 条件表达式解析失败：按默认分支执行并记录告警。

**界面原型描述**
菱形「O」图标，属性面板提供分支条件与默认分支配置。

### M4.4 事件网关

**功能说明**
事件网关等待消息/信号/定时器事件到达后决定路径，常用于外部系统回调、超时补偿等。

**业务规则**
- 事件网关出流只能连接事件节点；
- 接收到任一事件后取消其他等待分支；
- 超时未接收到事件可进入默认超时分支。

**前置条件**
流程运行到事件网关。

**后置条件**
根据事件类型激活对应分支。

**异常处理**
- 事件类型未匹配：保留在网关等待或进入超时分支。

**界面原型描述**
菱形「◇」图标，出流连接中间捕获事件。

### M4.5 条件表达式引擎

**功能说明**
支持字段值等于/不等于/大于/小于/包含/不包含、数值范围、正则匹配、复杂组合表达式。

**业务规则**
- 表达式语法支持 SpEL；
- 操作符支持 `==`、`!=`、`>`、`<`、`>=`、`<=`、`contains`、`matches`；
- 支持流程变量、表单字段、运行时计算结果。

**前置条件**
分支连线上需要配置条件。

**后置条件**
运行时计算表达式结果决定分支走向。

**异常处理**
- 表达式语法错误：设计时校验拦截；
- 运行时变量缺失：按 false 处理并记录日志。

**界面原型描述**
连线属性面板提供可视化条件编辑器与表达式输入框。

### M4.6 条件组合逻辑

**功能说明**
支持且条件（AND，同一组内条件同时满足）、或条件（OR，不同条件组任一满足）、条件组嵌套。

**业务规则**
- 同一条件组内默认 AND；
- 多个条件组之间默认 OR；
- 支持最多 3 层嵌套。

**前置条件**
存在多个条件。

**后置条件**
组合后的布尔结果作为分支命中依据。

**异常处理**
- 嵌套层级超限：提示“条件嵌套层级不能超过 3 层”。

**界面原型描述**
条件编辑器以树形结构展示条件组，支持新增子组、删除、排序。

### M4.7 条件优先级

**功能说明**
多分支并行时配置优先级顺序，高优先级分支条件满足时优先执行，避免多条件同时命中冲突。

**业务规则**
- 优先级为整数，数值越小优先级越高；
- 排他网关必须配置优先级；
- 同优先级时按画布顺序执行。

**前置条件**
排他/包容网关存在多条出流。

**后置条件**
按优先级顺序计算条件。

**异常处理**
- 优先级重复：保持原顺序，提示警告。

**界面原型描述**
属性面板分支列表中显示优先级排序手柄。

### M4.8 默认分支

**功能说明**
当所有条件均不满足时的兜底分支走向，确保流程不因条件遗漏而中断。

**业务规则**
- 排他网关必须配置默认分支；
- 默认分支条件表达式为空；
- 默认分支在条件计算阶段最后判断。

**前置条件**
流程存在条件分支。

**后置条件**
条件全假时激活默认分支。

**异常处理**
- 未配置默认分支：校验阶段报错。

**界面原型描述**
连线属性面板「默认分支」开关，开启后条件表达式禁用。

### M4.9 子流程支持

**功能说明**
支持内嵌子流程（同一流程定义内）和外部子流程调用（独立流程定义），父子流程可传递与同步数据。

**业务规则**
- 内嵌子流程共享父流程变量；
- 外部子流程通过输入/输出参数映射传递数据；
- 子流程异常可触发父流程边界事件。

**前置条件**
存在子流程定义或被调用流程定义。

**后置条件**
父流程等待子流程完成后再继续。

**异常处理**
- 外部子流程不存在：挂起父流程并告警。

**界面原型描述**
画布上子流程显示为可折叠矩形，内部可继续编排。

### M4.10 循环与回退

**功能说明**
退回后重新按流程定义流转（支持循环直到通过），管理员可前进/后退/跳跃到任意已执行节点。

**业务规则**
- 普通退回按配置规则重新流转；
- 管理员跳跃节点需记录审计日志；
- 循环次数可配置上限，防止死循环。

**前置条件**
流程实例已运行并存在历史节点。

**后置条件**
流程实例重新定位到目标节点。

**异常处理**
- 循环次数超限：强制终止流程并告警。

**界面原型描述**
流程实例管理台提供「节点跳跃」按钮，弹窗选择目标节点与原因。

## 用例描述

| 用例编号 | 用例名称 | 参与者 | 基本流程 |
|---|---|---|---|
| UC-M4-01 | 配置金额分支 | 流程管理员 | 添加排他网关 → 配置条件表达式与优先级 → 配置默认分支 → 保存 |
| UC-M4-02 | 并行会签 | 流程管理员 | 添加并行分裂 → 添加多个审批节点 → 添加并行汇聚 → 保存 |

## 状态机/时序图描述

### 排他网关判断时序

1. 流程引擎获取网关所有出流条件；
2. 按优先级排序；
3. 依次计算条件表达式；
4. 命中第一个 true 分支，激活该分支后续节点；
5. 全部 false 则激活默认分支；
6. 无默认分支则进入异常处理。
