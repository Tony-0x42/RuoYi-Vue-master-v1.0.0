# M3 审批节点配置

## 模块概述

审批节点是流程中最核心的任务节点。本模块定义审批人来源、审批方式、审批操作、退回规则、节点权限、去重机制等，确保流程能够准确、灵活地分配给正确的处理人。

## 功能架构图（文字描述）

```text
审批节点配置
    │
    ├── 审批人来源（指定/主管/角色/部门/自选/表达式）
    ├── 审批方式（会签/或签/依次审批）
    ├── 审批人范围与去重
    ├── 审批意见与操作
    ├── 退回规则
    └── 节点权限与国际化
```

## 详细功能需求

### M3.1 审批人来源设置

**功能说明**
审批节点可配置办理人来源，包括：指定成员、上级主管（N 级）、角色/岗位、部门负责人、发起人自选（范围限定）、动态表达式计算。

**业务规则**
- 至少配置一种来源；
- 指定成员支持多选；
- 动态表达式返回用户 ID 或用户 ID 列表；
- 表达式中可使用流程变量、发起人信息、表单字段。

**前置条件**
画布上已存在审批节点。

**后置条件**
流程运行到该节点时，根据来源解析出任务办理人。

**异常处理**
- 表达式解析失败：任务分配给流程管理员并记录告警；
- 未找到审批人：流程挂起，通知流程管理员。

**界面原型描述**
属性面板「审批人」Tab，提供来源类型单选，选择后展示对应配置项。

### M3.2 多级主管查找规则

**功能说明**
支持按组织架构查找直接上级、第 N 级上级、逐级向上直到某级、部门负责人链、虚线上级、项目 Leader。

**业务规则**
- N 级上级：N 为 1~10 的正整数；
- 虚线上级需在组织架构中维护虚线汇报关系；
- 项目 Leader 通过项目成员关系动态识别。

**前置条件**
组织架构数据完整。

**后置条件**
解析出具体用户 ID。

**异常处理**
- 主管链中断：按可找到的最高层级返回，或转交指定接替人。

**界面原型描述**
下拉选择查找规则，输入 N 级或目标层级。

### M3.3 审批方式

**功能说明**
支持会签（全员通过则通过，一人拒绝则拒绝）、或签（任一人通过则通过）、依次审批（按设定顺序串行审批）。

**业务规则**
- 会签：通过比例可配置（默认 100%），达到比例即通过；
- 或签：一人处理后任务结束；
- 依次审批：按列表顺序依次生成任务，前一人处理完后生成下一任务。

**前置条件**
已配置多个审批人。

**后置条件**
按方式生成任务实例并流转。

**异常处理**
- 会签中审批人离职：自动跳过或转交配置接替人。

**界面原型描述**
「审批方式」单选区域，选择会签时显示通过比例输入。

### M3.4 审批人范围限制

**功能说明**
限定可选审批人范围、排除发起人本人、排除已审批过的人、限定部门/角色范围。

**业务规则**
- 排除发起人：在发起人同时命中审批人时生效；
- 排除已审批人：避免同一人在多个节点重复审批；
- 范围限定支持部门、角色、用户组组合。

**前置条件**
审批人来源已配置。

**后置条件**
最终审批人列表经过过滤。

**异常处理**
- 过滤后审批人为空：触发管理员告警并按兜底规则处理。

**界面原型描述**
「范围限制」分组，提供复选框与范围选择器。

### M3.5 审批人去重机制

**功能说明**
相邻节点审批人为同一人时自动跳过（可配置）、全局审批人去重、连续相同审批人合并。

**业务规则**
- 相邻去重：默认开启，可在节点级关闭；
- 全局去重：在整个流程实例范围内生效；
- 合并后保留最后一个节点的权限与意见要求。

**前置条件**
存在多个审批节点。

**后置条件**
运行时动态判断是否跳过节点或合并任务。

**异常处理**
- 去重导致无审批人：进入异常分支或挂起。

**界面原型描述**
节点属性「去重设置」开关与单选。

### M3.6 审批意见配置

**功能说明**
配置意见必填/选填、审批意见模板预设、快捷意见标签（同意/不同意/需补充材料）、批量审批时意见处理。

**业务规则**
- 拒绝/退回操作时意见必须必填；
- 快捷意见可自定义；
- 批量审批时，意见可统一填写或分别填写。

**前置条件**
审批节点已配置。

**后置条件**
审批表单展示意见输入区域。

**异常处理**
- 必填意见为空：前端拦截提交。

**界面原型描述**
审批页面底部显示意见输入框与快捷标签；批量审批弹窗显示统一意见选项。

### M3.7 审批操作类型

**功能说明**
支持同意、拒绝、转交、加签、退回、撤回、终止、保存草稿、评论。

**业务规则**
- 同意/拒绝：标准流转；
- 转交：将任务移交给他人，原处理人不再参与；
- 加签：在当前节点实例临时增加前置/后置审批人；
- 撤回：发起人可在下一节点未处理前撤回；
- 终止：仅具有权限的用户可操作。

**前置条件**
用户拥有当前任务处理权限。

**后置条件**
根据操作更新任务与流程实例状态。

**异常处理**
- 无权限操作：返回 403；
- 操作冲突（并发）：基于数据库乐观锁重试或提示。

**界面原型描述**
审批页面顶部操作按钮区，根据节点权限配置动态显示/隐藏按钮。

### M3.8 退回规则配置

**功能说明**
支持退回上一步、退回指定节点（可跨节点）、退回发起人、退回后重新提交或直接回到退回人。

**业务规则**
- 退回上一步：默认支持；
- 退回指定节点：可配置可选节点范围；
- 退回发起人：流程回到开始节点后的第一个审批节点前；
- 退回后重新提交：被退回人修改后可选择按原路径重走或直返退回人。

**前置条件**
流程已运行到可退回节点。

**后置条件**
流程回退到目标节点，生成新的待办。

**异常处理**
- 目标节点已删除：提示“目标节点不存在，无法退回”。

**界面原型描述**
点击「退回」按钮弹出节点选择与退回方式选择。

### M3.9 节点操作权限控制

**功能说明**
配置当前节点处理人可执行的操作，如仅允许通过/拒绝，禁用转交/加签。

**业务规则**
- 默认开启同意/拒绝；
- 管理员可强制覆盖节点操作限制；
- 操作权限可与角色绑定。

**前置条件**
节点已保存。

**后置条件**
运行时根据权限渲染操作按钮。

**异常处理**
- 权限配置异常：使用默认权限。

**界面原型描述**
属性面板「操作权限」分组，复选框选择允许的操作。

### M3.10 节点名称与国际化

**功能说明**
节点名称自定义、支持多语言配置（中文/英文等）、支持动态变量插入（如「{申请人}的主管审批」）。

**业务规则**
- 节点名称 2~50 字符；
- 多语言以 key-value 形式存储；
- 变量语法 `{{variableName}}`，运行时替换。

**前置条件**
节点已创建。

**后置条件**
名称在画布、待办、流程图、通知中统一展示。

**异常处理**
- 变量不存在时保留原占位符。

**界面原型描述**
属性面板「基础信息」输入框，提供变量选择器。
