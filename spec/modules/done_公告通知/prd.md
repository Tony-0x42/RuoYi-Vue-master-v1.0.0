# 公告通知产品需求文档

## 1. 背景与目标

### 1.1 背景

企业重要通知通过邮件、微信群等多渠道发送，难以统一管理和确认员工是否已阅。

### 1.2 目标

建立统一的公告发布和阅读确认机制，确保重要信息传达到位。

## 2. 目标用户

| 用户角色 | 主要诉求 |
|---|---|
| 行政人员/HR | 发布公司制度、通知，确认员工已阅 |
| 管理层 | 发布重要决策，查看传达效果 |
| 普通员工 | 及时获取公司通知，确认已阅 |

## 3. 用户故事

- **作为** 行政人员，**我希望** 发布公告并设置阅读确认，**以便** 确保员工知晓。
- **作为** 员工，**我希望** 在一个地方查看所有公司公告，**以便** 不遗漏重要信息。
- **作为** 管理层，**我希望** 查看公告阅读率，**以便** 评估信息传达效果。

## 4. 业务流程

```text
起草公告
  ├── 设置可见范围
  ├── 选择是否需要阅读确认
  ├── 发布
  └── 推送通知
        └── 用户阅读并确认
              └── 生成阅读统计
```

## 5. 功能详述

### 5.1 公告分类

- 支持多级分类，如公司公告、部门通知。
- 分类支持启用/停用。

### 5.2 公告发布

- 支持富文本编辑和附件。
- 支持设置可见范围、有效期、阅读确认。
- 支持置顶。

### 5.3 公告展示

- 列表展示，置顶公告优先。
- 未读公告高亮显示。

### 5.4 阅读确认

- 需确认的公告，用户必须点击"已阅"。
- 管理员可催阅未确认用户。

### 5.5 阅读统计

- 展示已读/未读人数和名单。
- 支持导出未读名单。

## 6. 原型描述

### 公告列表页

- 顶部：分类筛选、搜索框。
- 列表：标题、发布时间、分类、已读/未读标识。
- 置顶公告固定在列表顶部。

### 公告详情页

- 标题、发布时间、分类、内容、附件。
- 底部：已阅按钮（如需确认）。

- [ ] 支持可见范围控制。

- [ ] 支持置顶和有效期设置。
- [ ] 支持阅读统计和未读名单导出。

## 8. 依赖与风险

### 依赖

- 消息通知中心；
- 组织架构数据；
- 文件存储服务。

### 风险

- 重要公告可能被用户忽略，需配合消息提醒和催阅机制。
