# 公告通知

## 模块概述

公告通知用于向组织内用户发布重要通知和制度文件，支持分类、置顶、阅读确认、附件和阅读统计。

## 功能架构图

```text
公告通知
    ├── 公告分类
    │   ├── 分类管理
    │   └── 分类权限
    ├── 公告发布
    │   ├── 公告编辑
    │   ├── 可见范围
    │   └── 阅读确认
    ├── 公告展示
    │   ├── 列表展示
    │   ├── 置顶公告
    │   └── 详情查看
    └── 公告统计
        ├── 阅读人数
        └── 未读名单
```

| 编号 | 功能 | 说明 |
|---|---|---|
| NO-1 | 公告分类 | 多级分类，支持权限控制 |
| NO-2 | 公告发布 | 编辑公告内容，设置可见范围和有效期 |
| NO-3 | 公告编辑/下架 | 维护公告状态 |
| NO-4 | 置顶公告 | 重要公告置顶展示 |

| NO-7 | 阅读统计 | 查看已读/未读人数和名单 |

## 详细功能需求

### NO-1 公告分类

**功能说明**
建立公告分类体系。

**业务规则**
- 分类层级最多 3 级；
- 分类编码唯一；
- 停用分类后其下公告不可见。

### NO-2 公告发布

**功能说明**
编辑公告并发布给指定范围用户。

**业务规则**
- 公告支持富文本和附件；
- 可见范围支持全部、指定部门、指定角色、指定用户；
- 支持设置有效期，到期自动下架；
- 支持选择是否需要阅读确认。

### NO-3 公告编辑/下架

**功能说明**
修改或下架已发布公告。

**业务规则**
- 仅发布人和管理员可编辑/下架；
- 编辑后重新发布需通知相关用户；
- 下架后用户不可见但历史可查询。

### NO-4 置顶公告

**功能说明**
将重要公告置顶展示。

**业务规则**
- 置顶公告最多 5 条；
- 置顶公告在列表顶部展示；
- 支持取消置顶。

### NO-5 阅读确认

**功能说明**
要求用户阅读后确认。

**业务规则**
- 需确认的公告，用户阅读后必须点击"已阅"；
- 未确认用户可发送催阅通知；
- 已确认记录保存 3 年。

### NO-6 附件

**功能说明**
公告支持上传附件。

**业务规则**
- 支持常见文档格式；
- 单公告附件总大小不超过 50MB。

### NO-7 阅读统计

**功能说明**
统计公告的阅读情况。

**业务规则**
- 展示已读人数、未读人数、阅读率；
- 支持导出未读人员名单；
- 仅发布人和管理员可查看统计。

## 用例描述

| 用例编号 | 用例名称 | 参与者 | 基本流程 |
|---|---|---|---|
| UC-NO-01 | 发布公告 | 行政人员 | 编辑公告 → 设置范围 → 发布 → 通知用户 |
| UC-NO-02 | 阅读公告 | 普通员工 | 收到通知 → 查看公告 → 点击已阅 |
| UC-NO-03 | 查看阅读统计 | 行政人员 | 进入公告详情 → 查看已读/未读统计 |

## 状态机/时序图描述

### 公告状态机

```text
[草稿] --发布--> [已发布]
[已发布] --过期--> [已过期]
[已发布] --下架--> [已下架]
[已过期] --重新发布--> [已发布]
```

## 数据模型

| 实体 | 说明 | 核心字段 |
|---|---|---|
| oa_notice_category | 公告分类 | id, parent_id, name, code, status, tenant_id |
| oa_notice | 公告 | id, category_id, title, content, scope_type, scope_ids, need_confirm, top, status, expire_time, tenant_id |
| oa_notice_attachment | 附件 | id, notice_id, file_name, file_path, size |
| oa_notice_read | 阅读记录 | id, notice_id, user_id, read_time, confirmed, tenant_id |
