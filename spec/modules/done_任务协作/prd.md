# 任务协作产品需求文档

## 1. 背景与目标

### 1.1 背景

团队任务分配和跟踪依赖 Excel 或线下工具，进度不透明，协作效率低。

### 1.2 目标

提供轻量级任务协作工具，支持任务全生命周期管理、看板视图和甘特图。

## 2. 目标用户

| 用户角色 | 主要诉求 |
|---|---|
| 项目经理 | 分配任务、跟踪进度、管理团队工作 |
| 团队成员 | 查看自己的任务、更新进度、协作沟通 |
| 管理者 | 查看团队任务完成情况 |

## 3. 用户故事

- **作为** 项目经理，**我希望** 分配任务给团队成员，**以便** 明确分工。
- **作为** 团队成员，**我希望** 在看板上查看任务状态，**以便** 了解工作重点。
- **作为** 管理者，**我希望** 查看任务完成率，**以便** 评估团队效率。

## 4. 业务流程

```text
创建任务
  ├── 分配负责人和参与人
  ├── 设置截止时间和优先级
  └── 跟踪状态
        ├── 更新进度
        ├── 评论协作
        └── 完成归档
```

## 5. 功能详述

### 5.1 任务管理

- 创建、编辑、删除任务。
- 设置负责人、参与人、起止时间、优先级。
- 支持子任务和标签。

### 5.2 看板视图

- 按状态分列展示任务。
- 支持拖拽改变状态。

### 5.3 甘特图

- 按时间轴展示任务进度。
- 支持周/月视图。

### 5.4 协作

- 任务内评论，支持 @ 提醒。
- 附件共享。

### 5.5 提醒

- 截止前提醒。
- 逾期任务高亮。

## 6. 原型描述

### 看板视图

- 横向为状态列：待处理、进行中、已完成。
- 每个任务为卡片，展示标题、负责人、截止时间、优先级。

### 任务详情页

- 标题、描述、负责人、参与人、时间、优先级。
- 子任务列表、评论、附件、操作日志。

## 7. 验收标准

- [ ] 支持任务 CRUD 和子任务。
- [ ] 支持看板视图和拖拽。
- [ ] 支持甘特图展示。
- [ ] 支持评论、附件、@ 提醒。
- [ ] 支持任务提醒和逾期高亮。
- [ ] 支持任务统计。

## 8. 依赖与风险

### 依赖

- 用户权限体系；
- 消息通知中心；
- 文件存储。

### 风险

- 任务数量大时看板性能可能下降，需做分页和懒加载。
