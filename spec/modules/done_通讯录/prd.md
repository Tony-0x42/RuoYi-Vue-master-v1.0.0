# 通讯录产品需求文档

## 1. 背景与目标

### 1.1 背景

员工查找同事联系方式时依赖微信群或口头询问，效率低且信息不准确。

### 1.2 目标

提供统一的企业通讯录，支持组织架构浏览、人员搜索和个人通讯录管理。

## 2. 目标用户

| 用户角色 | 主要诉求 |
|---|---|
| 普通员工 | 快速找到同事联系方式 |
| 新员工 | 了解公司组织架构 |
| 管理者 | 查看团队汇报关系 |

## 3. 用户故事

- **作为** 普通员工，**我希望** 快速搜索到同事联系方式，**以便** 及时沟通。
- **作为** 新员工，**我希望** 查看公司组织架构，**以便** 了解团队结构。
- **作为** 用户，**我希望** 保存常用联系人，**以便** 快速发起沟通。

## 4. 业务流程

```text
进入通讯录
  ├── 浏览组织架构
  ├── 搜索人员
  └── 查看详情
        ├── 拨打电话
        ├── 发送邮件
        └── 发送消息
```

## 5. 功能详述

### 5.1 组织架构

- 以树形结构展示部门层级。
- 点击部门展示该部门人员。

### 5.2 人员搜索

- 支持姓名、拼音、部门、岗位搜索。
- 搜索结果支持快速查看详情。

### 5.3 人员详情

- 展示基本信息和联系方式。
- 展示汇报关系。
- 支持一键沟通。

### 5.4 个人通讯录

- 添加常用联系人。
- 自定义联系人分组。

### 5.5 隐私控制

- 支持隐藏部分联系方式。
- 高管隐私保护。

## 6. 原型描述

### 通讯录页面

- 左侧：部门树。
- 中部：人员列表，展示头像、姓名、部门、岗位。
- 右侧：选中人员详情。

### 搜索页面

- 顶部搜索框。
- 搜索结果按人员/部门分组。

## 7. 验收标准

- [ ] 支持组织架构树浏览。
- [ ] 支持多维度人员搜索。
- [ ] 人员详情展示完整联系方式。
- [ ] 支持个人通讯录和分组。
- [ ] 支持隐私控制。

## 8. 依赖与风险

### 依赖

- 组织架构数据（sys_dept, sys_user, sys_post）；
- 用户权限体系。

### 风险

- 人员数据同步延迟可能导致通讯录信息不准确。
