# 会议管理产品需求文档

## 1. 背景与目标

### 1.1 背景

企业会议资源冲突频发，会议通知和纪要管理依赖线下工具，导致会议效率低下。

### 1.2 目标

建立统一的会议管理系统，实现会议室资源管理、会议预定、通知提醒、会议纪要的全流程线上化。

## 2. 目标用户

| 用户角色 | 主要诉求 |
|---|---|
| 普通员工 | 快速预定会议室，接收会议提醒 |
| 会议组织者 | 管理会议邀请、上传纪要、统计签到 |
| 行政人员 | 管理会议室资源，查看使用统计 |

## 3. 用户故事

- **作为** 普通员工，**我希望** 快速找到空闲会议室并预定，**以便** 高效安排会议。
- **作为** 会议组织者，**我希望** 自动通知参与人并上传纪要，**以便** 会议信息可追溯。
- **作为** 行政人员，**我希望** 管理会议室设备和查看使用率，**以便** 优化资源配置。

## 4. 业务流程

```text
预定会议
  ├── 选择时间和会议室
  ├── 冲突检测
  ├── 填写会议信息
  ├── 邀请参与人
  └── 发送通知
        └── 会前提醒
              └── 会议签到
                    └── 上传纪要
```

## 5. 功能详述

### 5.1 会议室管理

- 维护会议室名称、位置、容量、设备、图片。
- 支持启用/停用会议室。

### 5.2 会议预定

- 可视化选择时间段和会议室。
- 实时冲突检测，推荐可用会议室。
- 支持添加内部和外部参会人。

### 5.3 周期会议

- 支持按天/周/月重复预定。
- 支持修改单次或整个周期。

### 5.4 会议通知

- 预定、变更、取消时自动通知参与人。
- 会前 15 分钟默认提醒。

### 5.5 会议纪要

- 支持富文本编辑和附件上传。
- 可选择纪要可见范围。

### 5.6 会议签到

- 支持扫码签到和手动签到。
- 生成签到统计报表。

## 6. 原型描述

### 会议室视图

- 横向为时间轴，纵向为会议室列表。
- 已占用时段显示会议主题，空闲时段可点击预定。

### 会议详情页

- 展示会议主题、时间、地点、参与人、签到情况、纪要。

## 7. 验收标准

- [ ] 支持会议室资源管理和占用视图。
- [ ] 会议预定支持实时冲突检测。
- [ ] 支持周期会议和批量修改。
- [ ] 会议通知覆盖预定、变更、取消、会前提醒。
- [ ] 支持会议纪要上传和分发。
- [ ] 支持会议签到和出席统计。

## 8. 依赖与风险

### 依赖

- 会议室资源数据；
- 消息通知中心；
- 日程管理（会议同步）。

### 风险

- 高并发预定时可能出现冲突，需通过数据库唯一约束或分布式锁解决。
