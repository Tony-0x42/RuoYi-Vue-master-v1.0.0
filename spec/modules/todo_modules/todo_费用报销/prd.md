# 费用报销产品需求文档

## 1. 背景与目标

### 1.1 背景

传统报销流程繁琐，发票管理混乱，费用标准和预算控制滞后，财务审核工作量大。

### 1.2 目标

实现费用报销全流程线上化，支持发票识别、费用标准校验、预算控制和审批联动，提升报销效率。

## 2. 目标用户

| 用户角色 | 主要诉求 |
|---|---|
| 普通员工 | 快速提交报销，实时查看进度 |
| 审批人 | 清晰查看报销明细和发票 |
| 财务人员 | 高效审核、付款、生成凭证 |
| 管理者 | 掌握费用支出和预算执行 |

## 3. 用户故事

- **作为** 普通员工，**我希望** 拍照识别发票并自动填写报销单，**以便** 减少手工录入。
- **作为** 审批人，**我希望** 看到费用是否超标准，**以便** 快速决策。
- **作为** 财务人员，**我希望** 报销审批通过后自动生成付款申请，**以便** 提高工作效率。

## 4. 业务流程

```text
填写报销单
  ├── 上传发票（OCR 识别）
  ├── 填写费用明细
  ├── 费用标准校验
  ├── 预算校验
  ├── 提交审批
  ├── 多级审批
  ├── 财务审核
  └── 付款完成
```

## 5. 功能详述

### 5.1 发票管理

- 支持拍照/上传发票。
- OCR 自动识别发票信息。
- 发票验真和查重。

### 5.2 报销单

- 关联发票，自动填充金额。
- 填写费用明细和事由。
- 关联借款单冲销。

### 5.3 费用标准

- 按职级、城市、费用类型设置限额。
- 超标准提示或禁止提交。

### 5.4 预算控制

- 报销时校验预算余额。
- 审批通过后占用预算。

### 5.5 审批流转

- 通过流程引擎发起审批。
- 支持按金额、部门路由审批人。

### 5.6 统计分析

- 按部门、个人、费用类型统计。
- 预算执行率分析。

## 6. 原型描述

### 报销单填写页

- 顶部：报销单基本信息。
- 中部：费用明细列表，每行关联一张发票。
- 底部：合计金额、关联借款、提交按钮。

### 审批详情页

- 展示报销单信息、费用明细、发票图片、审批历史。

## 8. 依赖与风险

### 依赖

- 流程引擎；
- OCR 服务；
- 发票验真接口；
- 预算数据。

### 风险

- 发票识别准确率受限，需提供人工核对机制。
