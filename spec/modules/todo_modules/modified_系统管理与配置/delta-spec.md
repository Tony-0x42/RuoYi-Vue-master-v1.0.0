# 系统管理与配置（增量更新规范）

## 更新范围

现有 `modules/流程管理/M14-系统管理与配置.md` 面向「流程中台」设计，主要覆盖流程分类、流程权限、通知模板、工作日历、数据保留等流程相关配置。随着规范扩展为完整商用 OA 系统，M14 需扩展为通用系统管理能力，覆盖用户、角色、菜单、参数、字典、日志、应用、租户等基础运维能力。

本增量规范说明在保持现有 M14 流程相关能力不变的前提下，需要新增和修改的内容。

## 新增功能点

### SYS-1 用户管理

**功能说明**
管理系统用户账号，包括增删改查、导入导出、启用停用、重置密码、角色分配、部门调整。

**业务规则**
- 用户名、手机号、邮箱在租户内唯一；
- 支持批量导入用户（Excel 模板）；
- 停用用户后不可登录，已分配权限自动失效；
- 重置密码需二次确认，默认生成随机密码并通知用户；
- 支持配置用户直属上级和汇报关系。

**数据模型**
- 复用 `sys_user` 表；
- 扩展字段：直属上级 `leader_id`、入职日期 `entry_date`、离职日期 `leave_date`、状态 `status`。

### SYS-2 角色管理

**功能说明**
定义系统角色，分配菜单/按钮权限和数据范围。

**业务规则**
- 角色编码在租户内唯一；
- 支持角色继承（可选）；
- 数据范围支持：全部、本部门、本部门及下属部门、仅本人、自定义；
- 内置角色不可删除：超级管理员、系统管理员、普通用户。

**数据模型**
- 复用 `sys_role` 表；
- 复用 `sys_role_menu` 角色菜单关联表；
- 新增 `sys_role_data_scope` 角色数据范围表（如需细粒度控制）。

### SYS-3 菜单管理

**功能说明**
管理系统菜单和按钮权限，配置路由、图标、排序、组件路径。

**业务规则**
- 菜单类型：目录、菜单、按钮；
- 菜单权限标识（perms）全局唯一；
- 支持拖拽排序；
- 停用菜单后前端不再展示；
- 按钮权限用于前端按钮显隐和接口鉴权。

**数据模型**
- 复用 `sys_menu` 表。

### SYS-4 参数管理

**功能说明**
管理系统运行参数，支持热更新。

**业务规则**
- 参数键在租户内唯一；
- 支持字符串、数字、布尔、JSON 等多种类型；
- 修改后可通过缓存热更新，无需重启服务；
- 敏感参数（如密钥）需加密存储。

**数据模型**
- 新增 `sys_config` 或 `oa_system_param` 表：
  - id, param_key, param_value, param_type, description, encrypted, tenant_id, create_by, create_time, update_by, update_time。

### SYS-5 数据字典

**功能说明**
维护系统数据字典，如下拉选项、状态枚举等。

**业务规则**
- 字典类型编码唯一；
- 字典项支持排序、启用/停用；
- 已被业务数据引用的字典项不可删除；
- 支持按租户隔离或全局共享。

**数据模型**
- 新增 `sys_dict_type` 和 `sys_dict_data` 表。

### SYS-6 操作日志

**功能说明**
扩展 M14.4 操作日志为全系统操作日志，记录用户操作、配置变更、审批操作等。

**业务规则**
- 日志内容包括：操作人、IP、时间、操作模块、操作类型、操作对象、变更前后值；
- 支持按时间、模块、操作人、对象检索；
- 日志保留 3 年，到期归档；
- 敏感操作（如密码重置、权限变更）需单独标记。

**数据模型**
- 复用 M14.4 的 `bpm_audit_log` 或新增 `sys_oper_log` 表；
- 建议统一使用 `sys_oper_log` 作为全系统操作日志，`bpm_audit_log` 专注流程审计。

### SYS-7 登录日志

**功能说明**
记录用户登录、登出、登录失败等日志。

**业务规则**
- 记录登录时间、IP、浏览器、操作系统、登录状态（成功/失败）；
- 支持按用户、时间、IP 检索；
- 登录失败次数超过阈值可触发账号锁定。

**数据模型**
- 新增 `sys_login_log` 表：
  - id, user_id, username, ip, location, browser, os, status, msg, login_time, tenant_id。

### SYS-8 应用管理

**功能说明**
管理 OA 系统内各应用入口、图标、路由、权限。

**业务规则**
- 应用编码唯一；
- 应用与菜单关联；
- 支持配置应用可见范围；
- 应用状态：启用/停用。

**数据模型**
- 新增 `oa_application` 表：
  - id, code, name, icon, route, menu_id, scope_type, scope_ids, sort, status, tenant_id。

### SYS-9 租户管理

**功能说明**
多租户场景下管理租户信息和配置。

**业务规则**
- 租户编码唯一；
- 支持租户级参数、logo、主题配置；
- 超级管理员可管理所有租户，租户管理员仅管理本租户。

**数据模型**
- 复用或扩展现有租户表；
- 新增 `sys_tenant_config` 租户配置表。

## 修改点

### M14.3 管理员设置

**原内容**
设置流程管理员（管理指定分类/流程）、数据管理员（查看/导出数据）、超级管理员。

**修改后**
在原有流程管理员基础上，扩展为系统级角色管理：
- 超级管理员：拥有全部权限；
- 系统管理员：管理系统配置、用户、角色、菜单、日志；
- 流程管理员：管理流程分类和流程定义（保留原有语义）；
- 数据管理员：查看/导出全量数据（保留原有语义）。

### M14.4 操作日志

**原内容**
记录流程配置变更日志、审批操作日志、系统级操作日志。

**修改后**
- 流程相关审计继续由 `bpm_audit_log` 承担；
- 通用系统操作日志由 `sys_oper_log` 承担，覆盖用户、角色、菜单、参数、字典等配置变更。

### M14.8 多环境隔离

**原内容**
开发/测试/生产环境的流程配置与数据完全隔离。

**修改后**
扩展为全系统多环境隔离，包括流程配置、系统参数、应用配置、字典数据等。

## 兼容性说明

- 现有 M14 中流程分类、流程权限范围、通知模板、工作日历、数据保留策略等能力保持原有语义和表结构不变；
- 新增的通用系统管理能力独立建表，不破坏现有流程中台数据；
- `bpm_audit_log` 继续用于流程审计，新增 `sys_oper_log` 用于通用操作日志，两者可并行存在；
- 用户/角色/部门/岗位等基础表复用现有 `sys_*` 表，避免重复建设。
