# 通讯录

## 模块概述

通讯录提供企业组织架构浏览、人员搜索、联系方式查询和个人通讯录管理能力，是 OA 系统的基础用户查找入口。

## 功能架构图

```text
通讯录
    ├── 组织架构
    │   ├── 部门树
    │   └── 部门人员
    ├── 人员搜索
    │   ├── 按姓名
    │   ├── 按拼音
    │   └── 按部门/岗位
    ├── 人员详情
    │   ├── 联系方式
    │   └── 汇报关系
    └── 个人通讯录
        ├── 常用联系人
        └── 联系人分组
```

## 详细功能需求

### AB-1 组织架构树

**功能说明**
以树形结构展示企业部门层级。

**业务规则**
- 默认展开到二级部门；
- 点击部门展示该部门下人员；
- 部门排序按管理员配置。

### AB-2 人员搜索

**功能说明**
支持按多种条件搜索人员。

**业务规则**
- 支持姓名、拼音、手机号、部门、岗位搜索；
- 搜索结果按匹配度和部门排序；
- 支持搜索历史。

### AB-3 人员详情

**功能说明**
展示人员详细信息和联系方式。

**业务规则**
- 展示头像、姓名、部门、岗位、电话、邮箱、IM；
- 展示汇报关系（直属上级、下属）；
- 支持一键拨打电话、发送邮件/消息。

### AB-4 个人通讯录

**功能说明**
用户可添加常用联系人到个人通讯录。

**业务规则**
- 支持从组织架构添加；
- 支持移除常用联系人；
- 个人通讯录仅本人可见。

### AB-5 联系人分组

**功能说明**
用户可对个人通讯录进行分组。

**业务规则**
- 支持新建、重命名、删除分组；
- 一个联系人可属于多个分组。

### AB-6 最近联系人

**功能说明**
展示用户最近查看或沟通过的联系人。

**业务规则**
- 默认展示最近 20 条；
- 支持清空最近联系人。

### AB-7 隐私控制

**功能说明**
控制部分联系方式的可见性。

**业务规则**
- 手机号、邮箱等敏感信息可按角色或设置隐藏；
- 高管可设置隐私保护；
- 隐藏后仅允许通过系统内消息联系。

## 用例描述

| 用例编号 | 用例名称 | 参与者 | 基本流程 |
|---|---|---|---|
| UC-AB-01 | 查找同事 | 普通员工 | 进入通讯录 → 输入姓名 → 查看详情 |
| UC-AB-02 | 浏览部门架构 | 普通员工 | 进入通讯录 → 展开部门树 → 查看部门人员 |
| UC-AB-03 | 添加常用联系人 | 普通员工 | 查看同事详情 → 添加到常用联系人 |


## 数据模型

| 实体 | 说明 | 核心字段 |
|---|---|---|
| sys_dept | 部门 | 复用现有部门表 |
| sys_user | 用户 | 复用现有用户表 |
| sys_post | 岗位 | 复用现有岗位表 |
| oa_contact | 个人通讯录 | id, user_id, contact_user_id, group_id, tenant_id |
| oa_contact_group | 联系人分组 | id, user_id, name, sort, tenant_id |
| oa_recent_contact | 最近联系人 | id, user_id, contact_user_id, contact_time, tenant_id |
