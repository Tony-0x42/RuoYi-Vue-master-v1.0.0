# 考勤管理产品需求文档

## 1. 背景与目标

### 1.1 背景

企业考勤管理依赖人工统计，效率低、易出错，且难以与请假、加班、出差等流程联动。

### 1.2 目标

实现自动化考勤管理，支持多种打卡方式，与请假/加班/出差审批流程联动，自动生成考勤报表。

## 2. 目标用户

| 用户角色 | 主要诉求 |
|---|---|
| 普通员工 | 便捷打卡、申请请假/加班/出差、查看考勤 |
| HR | 配置考勤规则、查看统计报表、处理异常 |
| 部门经理 | 审批下属考勤申请、查看团队出勤 |

## 3. 用户故事

- **作为** 普通员工，**我希望** 用手机定位打卡，**以便** 方便记录出勤。
- **作为** 员工，**我希望** 请假审批通过后自动不计缺勤，**以便** 考勤准确。
- **作为** HR，**我希望** 自动生成考勤月报，**以便** 减少人工统计工作。

## 4. 业务流程

```text
设置考勤组和班次
  └── 员工每日打卡
        ├── 正常 → 计入出勤
        ├── 异常 → 申请补卡/请假/加班/出差
        │            └── 审批通过 → 更新考勤状态
        └── 月底自动生成考勤月报
```

## 5. 功能详述

### 5.1 考勤设置

- 创建考勤组，关联部门和班次。
- 配置打卡方式和地点。

### 5.2 打卡

- 支持 GPS、WiFi、考勤机、外勤打卡。
- 支持多次打卡和打卡记录查询。

### 5.3 考勤申请

- 请假、加班、出差、补卡均通过流程引擎审批。
- 审批结果自动同步考勤。

### 5.4 考勤统计

- 自动生成日报和月报。
- 展示迟到、早退、缺勤、请假、加班等数据。
- 支持导出 Excel。

### 5.5 假期余额

- 展示年假、调休等余额。
- 请假时自动扣减。

## 6. 原型描述

### 打卡页面

- 顶部：当前班次和打卡状态。
- 中部：打卡按钮和打卡记录。
- 底部：申请入口（请假、加班、出差、补卡）。

### 考勤统计页

- 日历展示每日出勤状态。
- 月报汇总卡片和明细列表。

## 7. 验收标准

- [ ] 支持 GPS/WiFi/考勤机/外勤多种打卡方式。
- [ ] 支持考勤组、班次、排班配置。
- [ ] 请假/加班/出差/补卡与流程引擎联动。
- [ ] 自动生成考勤日报和月报。
- [ ] 支持考勤异常处理和导出。
- [ ] 支持假期余额管理。

## 8. 依赖与风险

### 依赖

- 流程引擎；
- 定位服务/WiFi 识别；
- 工作日历；
- 组织架构。

### 风险

- 定位不准确可能导致打卡失败，需设置合理范围。
