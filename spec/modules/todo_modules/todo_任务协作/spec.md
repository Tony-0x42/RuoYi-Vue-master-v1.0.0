# 任务协作

## 模块概述

任务协作提供任务创建、分配、跟踪、协作能力，支持看板视图和基础项目管理，帮助团队高效协同工作。

```text

    ├── 任务管理
    │   ├── 创建任务
    │   ├── 任务分配
    │   ├── 状态更新
    │   └── 子任务
    ├── 视图
    │   ├── 列表视图
    │   ├── 看板视图
    │   └── 甘特图

    │   └── 提醒
    └── 统计
        ├── 任务完成率
        └── 逾期任务
```

## 详细功能需求

### TC-1 任务创建

**功能说明**
创建任务，填写标题、描述、负责人、参与人、起止时间、优先级等。

**业务规则**
- 任务标题必填；
- 负责人默认为创建人，可修改；
- 支持设置标签和关联项目/需求。

### TC-2 任务分配

**功能说明**
将任务分配给负责人和参与人。

**业务规则**
- 一个负责人，多个参与人；
- 分配后通知相关人员；
- 支持转交负责人。

### TC-3 任务状态

**功能说明**
维护任务生命周期状态。

**业务规则**
- 状态：待处理、进行中、已完成、已取消；
- 状态变更记录日志；
- 已完成任务可重新打开。

### TC-4 优先级

**功能说明**
设置任务优先级。

**业务规则**
- 优先级：紧急、高、中、低；
- 高优先级任务在看板中置顶。

### TC-5 子任务

**功能说明**
将任务拆解为多个子任务。

**业务规则**
- 子任务独立状态；
- 所有子任务完成后父任务可自动完成（可配置）。

### TC-6 看板视图

**功能说明**
以看板形式展示任务。

**业务规则**
- 支持按状态分列；
- 支持拖拽改变状态；
- 支持自定义看板列。

### TC-7 甘特图

**功能说明**
按时间轴展示任务进度。

**业务规则**
- 支持按周/月视图；
- 展示任务起止时间和进度；
- 支持任务依赖关系（可选）。

### TC-8 评论/附件

**功能说明**
任务内评论和附件共享。

**业务规则**
- 评论支持 @ 提醒；
- 附件按任务分组存储。

### TC-9 任务提醒

**功能说明**
任务截止前提醒相关人员。

**业务规则**
- 默认截止前 1 天提醒；
- 支持自定义提醒时间；
- 逾期任务高亮展示。

## 用例描述

| 用例编号 | 用例名称 | 参与者 | 基本流程 |
|---|---|---|---|
| UC-TC-01 | 创建任务 | 项目经理 | 填写任务信息 → 分配负责人 → 保存 |
| UC-TC-02 | 更新任务状态 | 负责人 | 进入看板 → 拖拽任务到进行中 → 更新进度 |
| UC-TC-03 | 查看任务统计 | 管理者 | 进入统计页 → 查看完成率和逾期任务 |

## 状态机/时序图描述

### 任务状态机

```text
[待处理] --开始--> [进行中]
[进行中] --完成--> [已完成]
[待处理] --取消--> [已取消]
[已完成] --重新打开--> [待处理]
```

## 数据模型

| 实体 | 说明 | 核心字段 |
|---|---|---|
| oa_task | 任务主表 | id, title, description, owner_id, project_id, priority, status, start_time, end_time, progress, tenant_id |
| oa_task_member | 任务成员 | id, task_id, user_id, role, tenant_id |
| oa_task_subtask | 子任务 | id, parent_id, title, status, tenant_id |
| oa_task_comment | 评论 | id, task_id, user_id, content, tenant_id |
| oa_task_attachment | 附件 | id, task_id, file_name, file_path, size |
| oa_task_board | 看板 | id, name, columns_json, tenant_id |
