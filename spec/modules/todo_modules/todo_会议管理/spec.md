# 会议管理

## 模块概述

会议管理为企业提供会议室资源管理、会议预定、会议通知、会议纪要的一体化能力，减少会议资源冲突，提升会议效率。

## 功能架构图

```text
会议管理
    ├── 会议室资源
    │   ├── 会议室管理
    │   └── 占用视图
    ├── 会议预定
    │   ├── 立即会议
    │   ├── 周期会议
    │   └── 冲突检测
    ├── 会议通知
    │   ├── 预定通知
    │   ├── 变更通知
    │   └── 会前提醒
    └── 会议纪要
        ├── 纪要上传
        ├── 纪要审批
        └── 纪要分发
```

## 详细功能需求

### MT-1 会议室管理

**功能说明**
维护会议室基础信息，包括名称、位置、容量、设备等。

**业务规则**
- 会议室编码唯一；
- 支持上传会议室图片；
- 停用后不可预定；
- 支持按楼层/区域分组。

### MT-2 会议预定

**功能说明**
用户选择时间、会议室、参与人完成会议预定。

**业务规则**
- 同一会议室同一时间只能被一个会议占用；
- 预定冲突时给出提示并推荐可用会议室；
- 参与人默认收到会议邀请；
- 支持添加外部参会人（仅邮件/短信通知）。

### MT-3 周期会议

**功能说明**
按固定周期重复创建会议。

**业务规则**
- 支持按天、周、月重复；
- 支持设置结束日期或重复次数；
- 修改周期会议可选择仅修改本次或全部。

### MT-4 会议通知

**功能说明**
通过消息通知中心发送会议相关通知。

**业务规则**
- 预定后即时通知参与人；
- 会议变更/取消时通知参与人；
- 会前默认 15 分钟提醒（可配置）。

### MT-5 会议纪要

**功能说明**
会后上传会议纪要，支持附件和分发。

**业务规则**
- 纪要支持富文本和附件；
- 可选择纪要可见范围；
- 重要纪要可设置审批流程。

### MT-6 会议签到

**功能说明**
会议现场签到，统计出席情况。

**业务规则**
- 支持扫码签到和手动签到；
- 签到时间记录精确到分钟；
- 生成签到统计报表。

### MT-7 占用查询

**功能说明**
按会议室和时间维度查询占用情况。

**业务规则**
- 支持日/周视图；
- 空闲时段可快速发起预定。

## 用例描述

| 用例编号 | 用例名称 | 参与者 | 基本流程 |
|---|---|---|---|
| UC-MT-01 | 预定会议 | 普通员工 | 选择时间 → 选择会议室 → 填写会议信息 → 发送邀请 |
| UC-MT-02 | 取消会议 | 会议组织者 | 进入会议详情 → 取消 → 通知参与人 |
| UC-MT-03 | 上传纪要 | 会议组织者 | 会议结束 → 上传纪要 → 分发 |

## 数据模型

| 实体 | 说明 | 核心字段 |
|---|---|---|
| oa_meeting_room | 会议室 | id, code, name, location, capacity, devices, status, tenant_id |
| oa_meeting | 会议 | id, title, start_time, end_time, room_id, organizer, content, status, tenant_id |
| oa_meeting_attendee | 参与人 | id, meeting_id, user_id, email, status, sign_in_time, tenant_id |
| oa_meeting_minutes | 会议纪要 | id, meeting_id, content, attachments, scope, status, tenant_id |
| oa_meeting_sign_in | 签到记录 | id, meeting_id, user_id, sign_in_time, type, tenant_id |
