# 八、部署架构

## 8.1 推荐架构

采用 **微服务 + 容器化** 部署，核心服务可独立扩容。

```text
                              负载均衡（Nginx/ALB）
                                     │
           ┌─────────────────────────┼─────────────────────────┐
           ▼                         ▼                         ▼
      管理控制台                  API 网关                     移动端/H5
           │                         │
           └──────────┬──────────────┘
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
   流程设计服务   流程运行时服务   表单服务
        │             │             │
        └─────────────┼─────────────┘
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
   组织同步服务   消息通知服务   监控/报表服务
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
      MySQL        Redis        MQ(Kafka/RocketMQ)
        │             │             │
        └─────────────┴─────────────┘
                      │
                   对象存储/OSS
                   日志/监控系统(ELK/Prometheus/Grafana)
```

## 8.2 服务划分

| 服务 | 职责 | 部署建议 |
|---|---|---|
| bpm-design-service | 流程/表单设计、版本管理、模板库 | 2+ 节点 |
| bpm-runtime-service | 流程实例、任务、审批操作、网关判断 | 4+ 节点 |
| bpm-form-service | 表单渲染、数据存储、联动计算 | 2+ 节点 |
| bpm-org-service | 组织架构同步、审批人解析 | 2+ 节点 |
| bpm-message-service | 消息通知、催办、回调推送 | 2+ 节点 |
| bpm-monitor-service | 监控、运营大盘、报表、异常检测 | 2+ 节点 |
| bpm-gateway | API 网关、鉴权、限流、路由 | 2+ 节点 |

## 8.3 数据存储

| 数据 | 存储 | 说明 |
|---|---|---|
| 流程定义/实例/任务 | MySQL | 主库，保证事务一致性 |
| 缓存 | Redis | 会话、Token、待办计数、流程定义缓存 |
| 消息队列 | Kafka/RocketMQ | 事件、回调、异步任务 |
| 文件 | OSS/MinIO | 附件、签名、模板、归档文件 |
| 日志 | ELK | 操作日志、审计日志、运行日志 |
| 报表/分析 | ClickHouse/Doris | 历史数据分析、运营报表 |

## 8.4 高可用设计

- 服务多节点部署，无状态设计；
- MySQL 主从 + Keepalived/MHA，RTO < 5 min；
- Redis 哨兵或集群模式；
- MQ 多副本；
- 定时任务通过 Elastic-Job/XXL-Job 集群调度。

## 8.5 网络与安全

- 内外网分离，DMZ 区部署网关；
- 数据库、缓存、MQ 位于内网，不暴露公网；
- 容器网络使用 Calico/Flannel；
- 通过 WAF 防护常见 Web 攻击。

## 8.6 多环境

| 环境 | 用途 | 数据 |
|---|---|---|
| 开发环境 | 开发联调 |  mock/测试数据 |
| 测试环境 | 集成测试/UAT | 脱敏业务数据 |
| 预发布环境 | 上线前验证 | 生产镜像 |
| 生产环境 | 正式运行 | 真实业务数据 |

## 8.7 扩展建议

- 初期可按单体或少数服务合并部署，降低运维复杂度；
- 随着流量增长，逐步拆分为独立微服务；
- 历史数据归档到 OLAP，避免主库性能瓶颈。
