# 四、非功能需求

## 4.1 性能要求

| 指标 | 目标值 | 说明 |
|---|---|---|
| 流程发起接口响应时间 | P99 < 500 ms | 单实例创建，不含外部回调 |
| 审批操作接口响应时间 | P99 < 300 ms | 同意/拒绝/转交等核心操作 |
| 流程图渲染时间 | < 2 s | 含 100 个节点的流程图 |
| 待办列表查询 | P99 < 500 ms | 单用户 1000 条待办以内 |
| 并发流程实例数 | ≥ 5000 TPS | 峰值场景 |
| 系统可用性 | 99.95% | 年度可用性目标 |
| 数据库写入延迟 | P99 < 100 ms | 同一机房内 |
| 报表查询 | 简单报表 < 3 s，复杂报表 < 30 s | 复杂报表走 OLAP |

### 容量规划

- 日均发起流程 10 万+；
- 峰值并发 5000+；
- 历史流程实例数据按年增长 3000 万+，需按 M14.7 策略归档。

## 4.2 安全要求

### 4.2.1 身份与访问

- 所有 API 必须鉴权，支持 OAuth 2.0 / OIDC / JWT；
- 支持 RBAC 与数据范围权限；
- 敏感操作需二次认证（如管理员干预、密钥查看）。

### 4.2.2 数据安全

- 传输层使用 TLS 1.2+；
- 存储层敏感字段 AES-256 加密；
- 数据库连接使用独立账号，最小权限原则；
- 数据脱敏按角色展示；
- 密钥由 KMS 管理，支持轮换。

### 4.2.3 审计合规

- 所有审批操作、配置变更、字段修改、登录登出留痕；
- 日志保留 3 年，不可篡改；
- 支持等保 2.0 三级要求。

### 4.2.4 防攻击

- API 网关限流、防重放、防 SQL 注入、防 XSS；
- 文件上传限制类型与大小，扫描恶意文件；
- 敏感接口访问频率限制。

## 4.3 可用性要求

- 核心流程服务多节点部署，支持无状态水平扩展；
- 数据库主从复制，支持故障自动切换（RTO < 5 min，RPO < 1 min）；
- 定时任务调度器集群化，避免单点；
- 消息队列保证事件不丢失；
- 降级策略：外部回调失败不影响主流程流转。

## 4.4 扩展性要求

- 微服务架构，核心服务可独立扩容；
- 流程节点、表单组件、连接器、校验规则支持插件化扩展；
- 多租户逻辑隔离，支持水平分库分表；
- 支持多云/混合云部署；
- 事件总线可外接 Kafka/RocketMQ 支撑大规模异步。

## 4.5 可维护性要求

- 代码遵循统一规范，关键路径有日志与监控；
- 配置集中管理，支持热更新；
- 接口文档自动生成并同步；
- 单元测试覆盖率 ≥ 60%，核心流程 ≥ 80%。

## 4.6 兼容性要求

- 浏览器：Chrome 90+、Edge 90+、Firefox 88+、Safari 14+；
- 移动端：iOS 12+、Android 8+；
- 支持 BPMN 2.0 XML 导入导出，兼容 Flowable/Camunda/Activiti 基础语义。
