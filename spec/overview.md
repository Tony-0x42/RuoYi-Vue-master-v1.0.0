# 一、文档概述

## 1.1 项目背景

随着集团业务快速发展，各业务系统（OA、ERP、CRM、HR、财务等）分别建设了独立的审批流程，导致：

- 流程能力重复建设，研发与维护成本高；
- 审批标准不统一，用户体验割裂；
- 流程数据分散，无法统一监控与优化；
- 组织架构变更时需要多处适配。

为此，集团决定建设 **「企业级统一流程引擎中台」**，作为全集团所有流程审批的中央处理平台。各业务系统通过标准 API 调用中台能力，无需各自建设流程能力。

## 1.2 项目目标

1. **统一流程能力**：为全集团提供统一的流程设计、运行、监控、优化能力。
2. **快速响应业务**：通过可视化流程/表单设计器，业务人员可在低代码甚至无代码环境下完成流程搭建。
3. **高可扩展性**：支持多租户、多系统接入、连接器扩展、Webhook 扩展。
4. **高可用与高安全**：满足金融级数据安全、审计合规、权限控制要求。
5. **数据驱动运营**：提供流程运营大盘、瓶颈分析、智能优化建议。

## 1.3 目标业务场景

**集团级 OA 审批中台**，覆盖 20+ 子公司、5 万员工，日均发起流程 10 万+，峰值并发 5000+，平均审批响应时间 < 1 秒。

## 1.4 目标用户

| 角色 | 定位 | 主要诉求 |
|---|---|---|
| 普通员工 | 流程发起人、审批人、抄送人 | 发起便捷、审批高效、消息及时 |
| 流程管理员 | HR/IT/业务 BP | 快速配置流程与表单、发布与版本管理 |
| 系统管理员 | 平台运维人员 | 租户管理、权限管理、日志审计、系统配置 |
| 外部系统开发集成人员 | 各业务系统后端开发 | 标准 API、完善文档、稳定回调 |
| 高层管理者 | 流程运营与决策者 | 数据报表、效率分析、风险监控 |

## 1.5 项目范围

### 1.5.1 纳入范围

- 流程设计器核心（M1）
- 表单设计器（M2）
- 审批节点配置（M3）
- 流程控制与网关（M4）
- BPMN 事件体系（M5）
- 高级审批功能（M6）
- 超时与催办（M7）
- 流程版本管理（M8）
- 中台集成与开放能力（M9）
- 组织架构集成（M10）
- 数据权限与安全（M11）
- 流程监控与运营（M12）
- 打印与签章（M13）
- 系统管理与配置（M14）

### 1.5.2 暂不纳入范围（后续迭代）

- AI 流程挖掘与预测分析；
- 区块链存证；
- 国际多语言全文翻译（首期支持中/英）。

## 1.6 术语表

| 术语 | 英文 | 说明 |
|---|---|---|
| 流程定义 | Process Definition | 流程的静态模型，包含节点、连线、规则等 |
| 流程实例 | Process Instance | 流程定义的一次运行实体 |
| 任务/待办 | Task | 流程实例中需要用户处理的工作项 |
| 节点 | Activity/Node | 流程图中的元素，如审批节点、网关、事件等 |
| 会签 | Counter-sign | 多人全部通过才通过，一人拒绝即拒绝 |
| 或签 | Any-sign | 任意一人通过即通过 |
| 加签 | Add-sign | 临时增加审批人 |
| 转交 | Transfer | 将任务移交给他人处理 |
| 委托 | Delegate | 按时间段授权他人代理审批 |
| BPMN | Business Process Model and Notation | 业务流程建模标注，当前采用 2.0 版本 |
| SLA | Service Level Agreement | 服务等级协议，用于时效承诺与监控 |
| Webhook | Webhook | 流程节点中对外发起的 HTTP 调用 |
| 连接器 | Connector | 预置或自定义的外部系统适配器 |
| 租户 | Tenant | 多租户场景中的逻辑隔离单元 |

## 1.7 参考标准

- BPMN 2.0（ISO/IEC 19510）
- OAuth 2.0 / OIDC
- RESTful API 设计规范
- 等保 2.0 三级安全要求
