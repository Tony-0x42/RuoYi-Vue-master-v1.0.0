## 一、菜单入口

登录系统后，在左侧菜单栏找到 **流程管理**，展开后包含：

- 流程分类
- 流程定义
- 变量定义
- 节点配置
- 流程实例
- 我的待办
- 我的已办

> 流程设计入口在“流程定义”列表中，点击操作列的 **流程设计** 按钮进入。

---

## 二、流程分类

流程分类用于对流程进行分组，便于后续检索和权限划分。

### 操作步骤

1. 进入 **流程管理 → 流程分类**。
2. 点击 **新增**，填写分类名称、分类编码、排序、备注。
3. 保存后，在创建流程定义时即可选择该分类。

### 注意事项

- 分类编码建议唯一，便于识别。
- 已使用的分类不建议直接删除，否则会导致关联的流程定义分类显示为空。

---

## 三、流程定义

流程定义是流程的元数据，包含流程名称、编码、分类等信息。

### 3.1 新增流程定义

1. 进入 **流程管理 → 流程定义**。
2. 点击 **新增**。
3. 选择 **流程分类**，填写 **流程名称**、**流程编码**，可填写备注。
4. 点击 **确定** 保存。

### 3.2 修改/删除

- **修改**：点击操作列 **修改** 按钮，可调整流程名称、编码、分类、备注。
- **删除**：点击操作列 **删除** 按钮，确认后删除该流程定义及其关联数据。

> 删除流程定义不会影响已经发起的流程实例的历史数据。

---

## 四、流程设计器

流程设计器基于 BPMN 2.0 标准，支持拖拽方式绘制流程图。

### 4.1 进入设计器

1. 进入 **流程管理 → 流程定义**。
2. 点击某条流程定义操作列的 **流程设计** 按钮，进入设计页面。

### 4.2 工具栏说明

左侧工具栏列出了可拖拽的 BPMN 组件，图标旁已标注中文含义：

| 图标 | 名称 | 说明 |
|------|------|------|
| ✋ | 抓手 | 拖动画布 |
| ☐ | 套索 | 框选多个元素 |
| ↔ | 空间 | 调整元素间距 |
| ↗ | 连线 | 手动连接两个节点 |
| ○ | 开始 | 流程起点，每个流程必须有且仅有一个 |
| ◎ | 中间 | 中间事件 |
| ⊙ | 结束 | 流程终点，必须有至少一个 |
| ◇ | 网关 | 分支/合并，可配置条件 |
| ▭ | 任务 | 用户任务，需要指定办理人 |
| ▭+ | 子流程 | 嵌入子流程 |
| 📄 | 数据对象 | 数据对象引用 |
| 🗄 | 数据存储 | 数据存储引用 |
| ▬ | 泳道 | 泳道/参与者 |
| ☐⧉ | 分组 | 注释分组 |

### 4.3 绘制流程

1. 从左侧拖拽 **开始** 到画布。
2. 拖拽 **任务** 到画布。
3. 点击 **任务**，在右侧 **属性面板** 中设置：
   - **名称**：如“经理审批”。
   - **办理人**：填写流程变量名，如 `approvalAssignee`。
   - **候选角色**（可选）：多个角色用逗号分隔。
   - **会审**（可选）：勾选后启用多实例会审，并填写集合变量名。
4. 拖拽 **结束** 到画布。
5. 使用自动出现的箭头或 **连线** 工具，将开始 → 任务 → 结束依次连接。
6. 选中 **连线**（SequenceFlow），可在属性面板填写 **条件表达式**，例如：

   ```
   ${leaveDays > 3}
   ```

### 4.4 保存与发布

- **保存模型**：将当前 BPMN 图形保存到数据库，不会生成新的流程版本。
- **发布流程**：将当前模型部署到 Flowable 引擎，生成一个新的流程版本，可用于发起流程实例。
- **重置**：重新加载最近一次保存的模型。

> 只有 **已发布** 的流程版本才能被发起。发布前请确保每个用户任务都配置了办理人变量。

---

## 五、流程版本控制

每次点击 **发布流程**，系统都会执行一次部署：

- Flowable 会生成新的 `ProcessDefinition`，版本号自动递增。
- 当前 `bpm_definition` 会指向最新的版本。
- 历史版本会保留在 `bpm_definition_version` 表中，可用于追溯。

发起流程时，系统总是使用 **当前最新已发布版本**。

## 六、流程变量配置

流程变量用于在运行时传递数据，例如办理人、请假天数等。

### 6.1 配置入口

1. 进入 **流程管理 → 流程定义**。
2. 点击操作列 **配置变量**。
3. 在变量列表页面新增、修改、删除变量。

### 6.2 常用变量

| 变量名 | 示例值 | 用途 |
|--------|--------|------|
| `approvalAssignee` | `1` | 指定用户任务的办理人（用户 ID） |
| `leaveDays` | `5` | 条件表达式判断 |
| `assigneeList` | `1,2,3` | 会审多实例集合 |

### 6.3 发起时传入变量

在 **发起流程** 弹窗中：

1. 选择流程定义。
2. 填写流程标题。
3. 点击 **添加变量**，输入变量编码和变量值。
4. 确认发起。

---

## 七、发起流程

1. 进入 **流程管理 → 流程实例**。
2. 点击 **发起流程**。
3. 选择要发起的 **流程定义**。
4. 填写 **流程标题**。
5. 添加需要的流程变量（如 `approvalAssignee=1`）。
6. 点击 **确定**。

发起成功后，流程进入第一个用户任务节点，办理人可在 **我的待办** 中看到该任务。

---

## 八、我的待办

进入 **流程管理 → 我的待办**，列表显示当前登录用户需要处理的流程任务。

### 8.1 办理任务

1. 点击操作列 **办理**。
2. 填写审批意见。
3. 点击 **确定**，任务向下流转。

### 8.2 退回任务

1. 点击操作列 **退回**。
2. 填写退回原因。
3. 点击 **确定**，流程回退到上一个节点。

> 退回需要流程中存在前置用户任务节点。如果是单节点流程，退回到开始事件可能无法生成新的待办。

### 8.3 拒绝任务

1. 点击操作列 **拒绝**。
2. 填写拒绝原因。
3. 点击 **确定**，流程实例直接结束。

### 8.4 指派任务

1. 点击操作列 **指派**。
2. 输入目标 **用户 ID**。
3. 点击 **确定**，任务办理人变更为指定用户。

### 8.5 特送（委派）任务

1. 点击操作列 **特送**。
2. 输入目标 **用户 ID**。
3. 点击 **确定**，任务被委派给该用户处理，原办理人仍可查看。

---

## 九、我的已办

进入 **流程管理 → 我的已办**，可查看当前登录用户已处理过的任务记录，包括审批意见、处理时间等。

---

## 十、流程跟踪

1. 进入 **流程管理 → 流程实例**。
2. 点击某条实例操作列的 **流程图** 按钮。
3. 系统会打开流程跟踪页面，展示该实例对应的 BPMN 流程图。

> 流程图会高亮当前处于激活状态的节点，便于查看流程运行到哪个环节。

## 十一、常见问题

### Q1：流程设计器左侧没有组件或组件不显示中文？

请确认前端服务已正常启动，并刷新了页面。组件中文标注由前端 CSS 实现，若未显示请按 `Ctrl + F5` 强制刷新。

### Q2：发布流程时报错“元素 bpmn:userTask 中不允许出现属性 assignee”？

说明办理人属性没有使用 Flowable 命名空间。请通过右侧属性面板设置办理人，系统会自动写入 `flowable:assignee`，不要直接手写 `assignee`。

### Q3：发起流程后，我的待办里没有任务？

常见原因：

- 用户任务没有配置办理人。
- 发起时传入的 `approvalAssignee` 变量值不是当前登录用户的用户 ID。
- 流程发布未成功，当前流程定义仍指向旧版本。

### Q4：流程分类为什么不显示？

请检查流程定义是否关联了流程分类，且分类数据未被删除。

### Q5：如何查看历史版本？

历史版本保存在数据库表 `bpm_definition_version` 中。后续可在前端增加“版本历史”功能进行展示。
